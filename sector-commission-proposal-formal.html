Project Proposal
板块场景提佣转化产品初步方案

基于现有板块场景探索提佣付费产品转化

当前板块功能主要解决“哪个板块涨了、热了”，但尚未充分承接更高价值的判断需求， 如板块是否属于真实热点、当前是否仍具参与价值，以及板块内应优先关注哪类个股。

本方案拟围绕“强度识别、阶段判断、个股分层”构建最小闭环， 探索在现有板块场景基础上延伸提佣付费产品的业务路径。

目标用户
板块场景活跃用户
已使用板块场景功能，具备进一步转化基础。
核心能力
强度 + 阶段 + 分层
用于验证免费场景向付费产品的承接能力。
业务价值
提佣转化提升
增强场景承接深度，并为提佣业务提供空间。
Problem Definition

一、核心问题定义

从用户视角看，现有板块场景与提佣付费转化之间存在一条尚未被充分承接的需求链路。

01

真假热点难识别

传统板块排序容易受到权重拉升、单票异动等因素干扰，用户难以快速区分真实热点与短期噪音。

02

参与时机难判断

即使识别出强势板块，用户仍缺乏对启动、发酵、高潮、退潮等阶段的清晰判断。

03

板块内选股难聚焦

热点板块成分股众多，不同阶段适合关注的股票类型不同，用户缺少明确分层视角。

Solution

二、产品解决方案

围绕用户判断链路，形成可与提佣业务结合的付费产品能力框架。

A

板块强度评分

综合涨停数量、资金表现、联动性等维度，对板块进行优先级识别。

B

板块阶段判断

将板块状态映射为启动、发酵、高潮、分歧、退潮等阶段，辅助用户识别机会期与风险期。

C

板块内个股分层

围绕龙头、中军、跟风、补涨等类型进行展示，提升板块内选股效率。

Value

三、产品价值

该方案的核心价值在于基于现有板块场景提升付费承接能力，而非单纯增加信息量。

用户价值

帮助用户更快识别真实热点、判断参与时机、聚焦重点个股，减少追高、误判和人工筛选成本。

业务价值

推动现有板块场景向高价值提佣产品延伸，增强产品差异化，并为业务转化提供抓手。

MVP Scope

四、MVP 范围

以最小可用闭环为原则，优先验证该场景是否具备提佣付费产品的延展空间。

模块 功能定义 价值体现
板块强度 对热点板块进行强弱排序与优先级识别 提升免费场景下的高价值内容感知
阶段判断 识别板块当前处于机会期还是风险期 提升用户对付费能力的需求感知
个股分层 围绕龙头、中军、跟风、补涨等类型进行展示 提升板块场景向付费产品的承接可能
Validation

五、验证逻辑

该方向的成立性主要取决于是否具备区分度、使用价值与转化潜力。

I

历史区分度

验证高分板块及阶段标签是否能够对历史热点表现形成有效区分。

II

使用深度

观察用户对板块详情、阶段信息和个股分层内容的关注度与停留时长。

III

主观认可度

验证用户是否将该能力视为有价值的付费产品，而非普通展示页面。

Conclusion

六、结论

该方向具备明确的用户需求基础、清晰的业务价值路径和可收敛的验证框架。

板块场景提佣转化产品的核心意义，在于基于现有板块场景延伸更高价值的付费能力。 该方向若验证成立，有望在不改变原有场景价值的基础上，进一步提升转化效率并打开提佣业务空间。
